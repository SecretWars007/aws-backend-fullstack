Billetera Digital - Endpoints y Flujos de Integración

1. Identificación y Autenticación del Dispositivo

Endpoints responsables del registro, autenticación y vinculación del dispositivo móvil con el servidor.

POST /v1/device-identify
Identifica y registra el dispositivo móvil en el servidor. Es el primer paso del flujo de onboarding.
Cabeceras HTTP
Content-Type: application/json; charset=UTF-8
Petición (Request Body)
{
  "event": 100,
  "notification_id": "string",        // ID de notificaciones push (FCM)
  "product": "string",                // Modelo y versión del dispositivo
  "reference": "string",              // Referencia (generalmente "00000000")
  "version": "string",                // Versión de la aplicación
  "certificate": false,               // Indica si se usa certificado digital
  "device_id": "string",              // ID único del dispositivo
  "device_type": "android",           // Tipo de SO (android, ios)
  "encrypted_device": "string",       // Identificador cifrado del dispositivo
  "send_id": "string"                 // ID para envío de notificaciones
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "key": "string",          // Clave de cifrado (pipe-separated numbers)
    "iv": "string",           // Vector de inicialización (pipe-separated numbers)
    "certified_id": 42,       // ID de certificación asignado
    "auth_token": "string"    // Token de autenticación para el dispositivo
  }
}
POST /v1/device-auth
Autentica el dispositivo usando las credenciales obtenidas en el paso anterior.
Petición (Request Body)
{
  "certificate": false,
  "device_id": "string",
  "device_type": "android",
  "encrypted_device": "string",   // Identificador cifrado del dispositivo
  "send_id": "string"
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "key": "string",          // Clave de cifrado renovada
    "iv": "string",           // Vector de inicialización renovado
    "certified_id": 7,        // ID de certificación
    "auth_token": "string"    // Nuevo token de autenticación
  }
}

2. Validación de Usuario y Catálogos

Endpoints para validar datos del usuario y obtener catálogos de valores (extensiones de documento).

POST /v1/document-extensions
Obtiene el catálogo de extensiones/regiones válidas para el documento de identidad.
Petición (Request Body)
{
  "auth_token": "string"   // Token de autenticación del dispositivo
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "extensions": [
      { "name": "La Paz", "extension": "LP", "type": "Q" },
      { "name": "Santa Cruz", "extension": "SC", "type": "Q" },
      { "name": "Extranjero", "extension": "EX", "type": "P" }
    ]
  }
}
POST /v1/users-validate
Valida los datos del usuario (documento, extensión, celular, email) contra el sistema central.
Petición (Request Body)
{
  "cellphone": "string",              // Número de teléfono
  "certified_id": 7,                  // ID de certificación del dispositivo
  "document_complement": "string",    // Complemento del documento (ej. "00")
  "document_extension": "string",     // Extensión (LP, SC, etc.)
  "document_number": "string",        // Número de documento
  "document_type": "Q",               // Tipo de documento
  "email": "string",                  // Correo electrónico
  "auth_token": "string"              // Token de autenticación
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "id": 1058432,               // ID interno del usuario
    "cic": "string",             // Código de Identificación Ciudadana
    "home_address": "string",    // Dirección domiciliaria
    "is_client": true,           // Indica si ya es cliente
    "is_married": false          // Estado civil
  }
}

3. Verificación Multifactor

Endpoints para la generación y validación de OTP (One-Time Password) y reconocimiento facial.

POST /v1/otp-generate
Genera y envía un código OTP al número de celular registrado.
Petición (Request Body)
{
  "cellphone": "string",
  "certified_id": 7,
  "otp": "string",         // Código OTP digitado por el usuario
  "auth_token": "string"
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "code": "OTP_VERIFIED",        // Estado de verificación
    "transaction_id": 9087654321,   // ID de transacción
    "date": "2026-05-10 17:35:00"
  }
}
POST /v1/face-recognition-init
Inicializa el proceso de reconocimiento facial, devolviendo la instrucción e imagen de referencia.
Petición (Request Body)
{
  "cellphone": "string",
  "certified_id": 7,
  "document_complement": "string",
  "document_extension": "string",
  "document_number": "string",
  "document_type": "Q",
  "email": "string",
  "auth_token": "string"
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "instruction": "Por favor mire directamente a la cámara...",
    "image": "base64_encoded_image_data",   // Implantilla de rostro de referencia (Base64)
    "session_id": "string"                   // Identificador de la sesión de reconocimiento
  }
}
Nota: El campo image contiene una imagen en formato Base64 (JPEG) que debe ser mostrada al usuario como guía.
POST /v1/face-recognition-valid
Envía la selfie del usuario para validación contra la implantilla de referencia.
Petición (Request Body)
{
  "cellphone": "string",
  "certified_id": 7,
  "document_complement": "string",
  "document_extension": "string",
  "document_number": "string",
  "selfie": "base64_encoded_photo"   // Selfie capturada en Base64
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "OK",
  "data": {
    "code": "FACE_VERIFIED",         // Verificación facial exitosa
    "transaction_id": 8837461029,
    "date": "2026-05-10 17:46:00"
  }
}

6. Operaciones Transaccionales

Endpoints para ejecutar recargas de celular, transferencias bancarias y consultar movimientos.

POST /v1/recharge-entel  |  /v1/recharge-tigo  |  /v1/recharge-viva
Ejecuta una recarga de saldo para el operador específico (Entel, Tigo o Viva).
Petición (Request Body) - Ejemplo para Entel
{
  "account_type": "BIL",
  "bill_name": "GUSTAVO PARKER",
  "bill_nit": "1234567",
  "code": "75797978",              // Número de celular a recargar
  "currency": "BOL",
  "origin_account": "70012345",
  "total_amount": 10.00,
  "certified_id": 7,
  "pin": "1111",
  "private_token": "string",
  "auth_token": "string"
}
Respuesta Exitosa (200 OK)
{
  "state": 0,
  "message": "Recarga Entel realizada exitosamente",
  "data": {
    "code": "REC-998877",
    "transaction_id": 1029384756,
    "date": "2026-05-11 18:26:56"
  }
}
Nota: Para Tigo el campo code se reemplaza por destination_account y se incluye id del monto. Para Viva la estructura es similar a Entel.

7. Códigos de Respuesta Comunes

CódigoSignificadoDescripción
0OKOperación exitosa.
-1Error GenéricoError no especificado en el servidor.
-2Validación FallidaLos datos enviados no superan las validaciones de negocio.
-3No AutorizadoToken de autenticación inválido o expirado.
Consideraciones Adicionales:
  • Todos los endpoints requieren la cabecera Content-Type: application/json.
  • La URL base ha sido omitida intencionalmente por requerimiento.
  • El campo auth_token se obtiene en /v1/device-identify y se renueva en /v1/device-auth.
  • El campo private_token se obtiene después del login exitoso en /v1/sign-in y es requerido para endpoints autenticados.
  • Las imágenes (reconocimiento facial) se transmiten en formato Base64.
